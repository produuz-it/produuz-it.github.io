# Analysis

The "analysis" builds the information needed for the panel viewer based on the uploaded files. The following are the main steps:

  • If there are full project IFCs, a splitting process is accomplished. Each unique panel will result in a new IFC file. Those files can be downloaded.

  • The system collects all the drawings, details, docs, and panel IFCs (discarding full project IFCs).

  • A list of design names is built based on IFC filenames and drawings filenames.

  • All the docs are linked to the design, so they are always available.

  • Drawings are scanned to find detail names. Found detail names are linked to the design.

  • IFCs are scanned looking for detail names. Found names are linked to the design.

  • IFC geometry is analyzed to find rotation&traslation matrices (orientation of the panel in the world coordinates system).

  • IFC parts are parsed to the main parts database.

# Parts parsing to database

Each part of the IFC will be analyzed, generating drawings and metadata which is saved in the database. This database table will bring fast access to the information when the user clicks on a part in the 3D viewer.

Drawings are only generated for "wood" and "plate" elements. So, if you don't have a good IFC Parsing Setup, you may find your panel with some missing drawings.
